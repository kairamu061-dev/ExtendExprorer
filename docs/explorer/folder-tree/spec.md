# folder-tree 仕様

## 機能一覧

| # | 機能名 | 説明 |
|---|--------|------|
| 1 | ツリー表示 | ウィンドウ左側に固定幅（240px）のパネル。ルートは「ホーム」＋準備完了ドライブ（`DriveInfo.IsReady == true`）を列挙順に表示 |
| 2 | 遅延展開 | シェブロンクリックで初めてサブフォルダを列挙して子ノードを作る。ファイルは表示しない。2 回目以降の展開は再列挙しない |
| 3 | クリック移動 | ノード本体のクリック（Invoke）でアクティブペインのアクティブタブが該当パスへ移動（履歴に積む＝戻れる） |
| 4 | 折りたたみ | ヘッダの「◀」でパネルを幅 28px のストリップへ折りたたみ。「▶」で再展開。状態はウィンドウ生存中のみ保持 |
| 5 | 薄灰色表示 | 隠し・システム属性のフォルダは Opacity 0.55（file-list と同一ルール） |

## 画面・操作フロー

1. 起動 → 左側にツリー（ホーム＋ドライブ）が表示される
2. ノードのシェブロンをクリック → サブフォルダが遅延読込されて展開
3. ノード名をクリック → アクティブペインのアクティブタブがそのフォルダへ移動
4. ヘッダ「◀」をクリック → パネルが細いストリップに折りたたまれ、一覧領域が広がる

## 各画面 / 状態の詳細

### ツリーパネル（展開時）

- **表示内容**: ヘッダ（「フォルダー」ラベル＋◀ボタン）、TreeView（フォルダアイコン＋フォルダ名）
- **操作**: シェブロン=展開/折りたたみ、ノード Invoke=移動、◀=パネル折りたたみ
- **遷移先**: アクティブタブのフォルダ移動（ペイン構成は変わらない）

### ツリーパネル（折りたたみ時）

- **表示内容**: 幅 28px のストリップに「▶」ボタンのみ
- **操作**: ▶=パネル再展開
- **遷移先**: なし

## エラーケース

| 条件 | メッセージ / 挙動 |
|------|------------------|
| アクセス不可フォルダを展開 | 子ノード 0 件として展開（エラーダイアログは出さない）。シェブロンは消える |
| 展開後にフォルダが削除されていた | 同上（列挙失敗を空扱い） |
| ノード Invoke 先が存在しない | タブ側の既存エラー処理（「フォルダが見つかりません」表示）に委ねる |
| ドライブが 1 台も準備完了でない | ホームのみ表示 |

## 未対応ケース

- タブ移動へのツリーの追従（選択・自動展開）
- ツリーの手動更新（リフレッシュ）・ファイルシステム変更の監視
- ツリー幅のドラッグ変更（固定 240px）
- 折りたたみ状態のセッション保存（session サブ項目で扱う可能性あり）
