# folder-tree 開発メモ

## 実装上の判断

| 判断内容 | 理由 |
|----------|------|
| `TreeViewNode` を code-behind で組む方式ではなく、`ItemTemplate`（ルート要素 `TreeViewItem`）＋`ItemsSource` のデータバインディング方式 | `{x:Bind}` で AOT 互換を保てる。遅延展開は `HasUnrealizedChildren`（OneWay）＋`TreeView.Expanding` で実現できる |
| ツリーは全ペイン共有でウィンドウ左端に 1 つ | 要望文言「画面左側に」に一致（エクスプローラー同様）。ペインごとに持つとメモリも増える |
| 移動先はアクティブペインのアクティブタブ（新タブを開かない） | エクスプローラーと同じ操作感。履歴に積むので「戻る」で復帰できる |
| 展開失敗（アクセス拒否・消滅）は「子なし」扱いでダイアログなし | spec のエラーケース。ツリー操作の途中でモーダルを出さない |
| `DriveInfo.GetDrives()`/`IsReady` は `Task.Run` 内で評価 | ネットワーク/リムーバブルドライブでブロックし得るため UI スレッドで回さない |
| 折りたたみは Width 切替＋Visibility のみ（ツリーを破棄しない） | 再展開時に展開状態を維持。ノードは遅延読込なので保持コストは小さい |

## 発生した問題と対処

| 問題 | 対処 |
