# shell-icons 仕様

## 機能一覧

| # | 機能名 | 説明 |
|---|--------|------|
| 1 | シェルアイコン表示 | 一覧の各行に、シェルの関連付けアイコン（小アイコン 16px）を表示。エクスプローラーと同じ見た目 |
| 2 | 拡張子キャッシュ | 通常ファイルは拡張子単位で 1 回だけ取得して共有。フォルダは汎用フォルダアイコン 1 種 |
| 3 | 個別アイコン | .exe / .ico / .lnk は埋め込みアイコンが個別のため、フルパス単位で取得・キャッシュ |
| 4 | 非同期読込 | アイコン取得はバックグラウンド。完了までは従来グリフを表示し、完了後に差し替える |

## 画面・操作フロー

1. フォルダを表示 → 各行はまず従来グリフで即時表示される
2. 画面内の行から順にアイコンが解決され、シェルアイコンへ差し替わる（キャッシュ済みなら即時）

## 各画面 / 状態の詳細

### 一覧の行アイコン

- **表示内容**: 16×16 のシェルアイコン。未解決/取得失敗時は従来グリフ（フォルダ  / ファイル ）
- **操作**: なし（表示のみ）
- **遷移先**: なし

## エラーケース

| 条件 | メッセージ / 挙動 |
|------|------------------|
| SHGetFileInfo が失敗（アイコンなし） | 従来グリフのまま表示し続ける |
| アイコンのピクセル変換に失敗 | 同上（例外は握りつぶす） |
| ファイルがアイコン取得前に削除された | 同上（拡張子キャッシュは属性ベース取得のためディスクに触れない） |

## 未対応ケース

- folder-tree ノードのシェルアイコン化（将来対応）
- オーバーレイ（ショートカット矢印等）・サムネイル
- 関連付け変更のリアルタイム反映（キャッシュはプロセス生存中固定。再起動で反映）
