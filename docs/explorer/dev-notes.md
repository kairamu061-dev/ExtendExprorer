## 実装上の判断

<!-- なぜその実装にしたかを記述する -->

| 判断内容 | 理由 |
|----------|------|
| 技術スタックを Electron から WinUI 3 (C#/.NET 8) に変更（2026-07-04） | ユーザ方針「動作の軽さとメモリ使用量の節約」。Tauri / Wails / WinUI 3 / Electron を比較提示し、ユーザが WinUI 3 を選択 |
| タブ UI に WinUI 標準 TabView を採用 | 追加ボタン「＋」・クローズ・ホイールクリックを標準装備し、自前実装より軽くて確実 |
| 配布は当面 unpackaged（MSIX 化は将来検討） | 開発初期の反復を優先。パッケージングは配布段階で判断 |
| DI コンテナを使わず App.OnLaunched の合成ルートで手動注入（2026-07-05） | 現状サービス 2 つ程度でコンテナは過剰。増えたら Microsoft.Extensions.DependencyInjection 導入を再検討 |
| ツールバーに暫定のパス表示 TextBlock を設置（2026-07-05） | file-list 単体の動作確認に現在位置の表示が必要なため。address-bar 実装時に置き換える |
| 隠し・システム属性は Opacity 0.55 の減光で薄灰色表現（2026-07-05） | ユーザ要望「常時表示のまま色だけ変えたい」。テーマ（ライト/ダーク）どちらでも自然に沈むため固定色でなく不透明度を採用 |
| ソート状態はフォルダ移動で既定（名前昇順）にリセット（2026-07-05） | ユーザ要望。タブ複製時のソート引き継ぎも「しない」で統一（フォルダ単位の一時状態と定義） |
| CI に Native AOT ビルドを実験追加（2026-07-05） | メモリ使用量削減の検証。通常ビルド（147MB 起動時）に対する効果を実測で判断し、良ければ AOT を正式採用 |
| Native AOT を正式ビルドに昇格（2026-07-08 検証合格を受け 2026-07-11 入替） | BUG-001 解消確認済み。起動時メモリ 141.2→111.9MB（約21%減）・配布 zip 63→31MB。成果物名: `ExtendExprorer-win-x64`=AOT、`-jit`=切り分け用フォールバック |

## 設計からの変更点

<!-- 設計書との差分と理由を記述する -->

| 変更内容 | 理由 |
|----------|------|
| tabs: タブ操作コマンドを PaneViewModel でなく MainViewModel の `DuplicateActiveTab` / `CloseTab` に集約（2026-07-05） | 最終タブ→ペインクローズ規則がレイアウト木の操作でありもともと MainViewModel 委譲だったため、分割せず一箇所にまとめた |
| tabs: TabViewItem にコンテンツを持たせず、タブ下の共有領域 1 つ（ツールバー＋FileListView）をアクティブタブに差し替える方式（2026-07-05） | タブ数分の ListView を生成しないことでメモリ増を抑える（軽量方針）。切替は PaneView の code-behind で同期 |
| pane-split: GridSplitter（CommunityToolkit Sizers）でなく自作 SplitterBar（Border 派生 約80行）を採用（2026-07-11） | AOT 起動問題（BUG-001）を経て外部 UI ライブラリの AOT 互換リスクを避けたい。必要機能はドラッグとクランプのみで自作が安価 |
| pane-split: アクティブペインの枠線は太さ固定 1px・色のみ切替（仕様は 2px/1px） | 太さ変更はレイアウトが 1px ずれて見えるため。色（#0078D4 / #D0D0D0）で十分判別できる |
| pane-split: 分割・クローズ時はレイアウト木全体を再構築（設計どおり単純さ優先） | ペイン上限 8 なら再構築コストは無視できる。PaneView の状態は ViewModel 側にあるため再構築で失われない |

## 今後の課題

<!-- 現状の制限・将来対応すべき事項を記述する -->

- タブごとの履歴（戻る/進む）はセッション復元対象外（初期版はパスのみ）
- 大量ファイル（1 万件超）時の性能は仮想化 ListView で見込みだが、実測は未実施
- メモリ使用量の計測基準（起動直後/System32 後）: 雛形 114MB → file-list 147/182MB → tabs 142.6/200.9MB(タブ操作後) → **AOT 正式採用後 111.9/157.7MB**（2026-07-08 計測）。参考ツール（Tablacus, Win32 ネイティブ）は約 75MB で、WinUI 3 の下限的にこれ以上の大幅削減は困難。今後は 111.9MB を基準に増加を監視する
- タブクローズ後のメモリ回復（リーク有無）の長時間シナリオ計測は未実施（検証セッションが次回以降に予定）
- ユニットテストプロジェクトが未整備（初期版は手動 E2E のみ。ViewModel はサービス注入で分離済みのため導入は容易）
