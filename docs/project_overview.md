# プロジェクト概要

## 目的・背景

Windows 11 標準のエクスプローラーはタブ機能を持つが、画面分割（複数ペイン同時表示）や柔軟なセッション復元ができない。
本プロジェクト「ExtendExprorer」は、Tablacus Explorer のようなマルチタブ・マルチペイン型のファイルエクスプローラーを Windows 11 向けに提供し、複数フォルダを同時に見ながらのファイル整理作業を効率化する。

## スコープ

### 作るもの

- タブ型ファイルエクスプローラー（1 ペインに複数タブ）
- 画面分割（複数ペインの同時表示）
- 各ペインのフォルダパス（アドレスバー）表示
- アプリ終了時の状態保存と、次回起動時の復元（タブ・ペイン構成・各タブのパス）
- 「＋」ボタンによるタブの複製（現在のタブと同じパスで新規タブを開く）
- フォルダ内容の一覧表示と基本的なナビゲーション（ダブルクリックで移動、上へ、戻る/進む）

### 作らないもの

- ファイルのコピー/移動/削除/リネームなどの編集操作（初期スコープ外、将来検討）
- ネットワークドライブ・クラウドストレージの特別対応
- アドオン/プラグイン機構
- Windows 以外の OS への正式対応

## 技術スタック概要

| 技術 | 用途 | 選定理由 |
|------|------|----------|
| Electron | デスクトップアプリ基盤 | Windows 11 で動作し、開発環境（Linux devcontainer / Node.js）でそのまま開発・動作確認できる |
| Node.js (main process) | ファイルシステムアクセス・状態永続化 | `fs` API でフォルダ列挙が容易。devcontainer に導入済み |
| HTML/CSS/TypeScript (renderer) | UI（タブ・ペイン・ファイル一覧） | 柔軟なレイアウト（分割・タブ）を実装しやすい |
| electron-builder | Windows 向けパッケージング | Windows インストーラ/ポータブル exe の生成 |

## 全体アーキテクチャ

- **Main プロセス（Node.js）**: ウィンドウ管理、ファイルシステム列挙 API、セッション状態の保存/読込（JSON ファイル）
- **Renderer プロセス（UI）**: ペインレイアウト管理、タブ管理、アドレスバー、ファイル一覧表示
- **IPC**: Renderer → Main へフォルダ列挙・状態保存を要求（contextBridge 経由、nodeIntegration 無効）

```
[Renderer UI]  ペイン / タブ / アドレスバー / ファイル一覧
      │ IPC (contextBridge)
[Main]         fs 列挙 API ・ セッション保存(JSON) ・ ウィンドウ管理
```

## 制約

- 実行ターゲットは Windows 11（開発・CI は Linux devcontainer 上で行い、UI 動作確認は Electron を Linux 上で起動して行う）
- 開発環境には Node.js のみ導入済み（.NET や Rust 等は使わない）
- ファイル編集操作を持たないため、初期版は読み取り専用アクセスのみ

## UI デザイン方針

### コンセプト

Tablacus Explorer 風の高密度・実用重視 UI。装飾より情報量と操作効率を優先する。
（参考スクリーンショット: 4 ペイン分割、各ペインにタブバー＋アドレスバー＋ファイル一覧）

### カラーパレット

| 用途 | カラーコード | 説明 |
|------|-------------|------|
| プライマリ | #0078D4 | Windows 11 アクセントブルー（選択・アクティブタブ） |
| セカンダリ | #E5F1FB | 選択行・ホバーの背景 |
| 背景 | #FFFFFF | ペイン背景 |
| テキスト | #1F1F1F | 基本テキスト |
| エラー | #C42B1C | エラー表示 |

### タイポグラフィ

- フォント: "Segoe UI", "Yu Gothic UI", sans-serif
- 基本サイズ 12–13px（ファイル一覧は高密度表示）

### コンポーネント方針

- タブ・ボタンはフラットデザイン、境界線は 1px の淡いグレー
- ファイル一覧は詳細表示（名前・更新日時・種類・サイズ）のテーブル形式
- ペイン境界はドラッグ可能なスプリッター
