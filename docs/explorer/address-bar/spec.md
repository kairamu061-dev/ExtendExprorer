# address-bar 仕様

## 機能一覧

| # | 機能名 | 説明 |
|---|--------|------|
| 1 | パス表示 | ペインのアクティブタブの現在パスをテキストボックスに表示。タブ切替・移動で自動更新 |
| 2 | パス入力移動 | テキストボックスを編集して Enter → 存在確認のうえアクティブタブを移動（履歴に追加） |
| 3 | 編集キャンセル | Esc または フォーカス喪失で編集を破棄し、現在パスの表示に戻す |
| 4 | エラー表示 | 存在しないパスなら入力欄下に「パスが見つかりません」を 3 秒表示し、移動しない |

## 画面・操作フロー

1. アドレスバーをクリック → 全選択状態で編集モードに入る
2. パスを入力して Enter → `IFileSystemService.ListAsync` で検証 → 成功ならタブ移動、失敗ならエラー表示
3. Esc → 編集破棄、現在パス表示に戻る

## 各画面 / 状態の詳細

### 表示状態

- **表示内容**: アクティブタブのフルパス（Windows 形式 `C:\...`）
- **操作**: クリックで編集状態へ
- **遷移先**: 編集状態

### 編集状態

- **表示内容**: 入力中の文字列（初期値は現在パス、全選択）
- **操作**: Enter（確定）、Esc / blur（キャンセル）
- **遷移先**: 表示状態

## エラーケース

| 条件 | メッセージ / 挙動 |
|------|------------------|
| 存在しないパス | 「パスが見つかりません」を表示、移動しない |
| ファイルのパスを入力 | 親フォルダに移動し、該当ファイル行を選択（初期版は単に親フォルダへ移動でも可） |
| アクセス不能なパス | 移動は行い、一覧側のエラー表示に委ねる |

## 未対応ケース

- 環境変数（%USERPROFILE% 等）の展開
- UNC パス（\\server\share）の特別扱い（fs がエラーを返せばエラーケース扱い）
