# context-menu 仕様

## 機能一覧

| # | 機能名 | 説明 |
|---|--------|------|
| 1 | 項目メニュー | 一覧の項目上で右クリック → その項目を対象にしたシェルのコンテキストメニューをカーソル位置に表示。選択も右クリックした項目に移す |
| 2 | 背景メニュー | 項目のない空白部分で右クリック → 表示中フォルダの背景メニュー（新規作成・貼り付け・表示・プロパティ等）を表示 |
| 3 | コマンド実行 | メニュー項目のクリックでシェルの既定動作を実行（開く/コピー/削除/名前の変更※/プロパティ/送る/…）。実行はシェル任せで、結果の一覧反映は F5 相当の再読込に依存しない（自動更新は未対応） |
| 4 | 動的サブメニュー | 「送る」「プログラムから開く」等の遅延生成サブメニューを正しく展開（IContextMenu2/3 の `HandleMenuMsg` 転送） |

※「名前の変更」はシェルがエクスプローラーのインライン編集を前提とするため動作しない可能性がある（エラーケース参照）

## 画面・操作フロー

1. 一覧の項目を右クリック → 項目が選択され、カーソル位置にシェルメニューが表示される
2. メニュー項目をクリック → シェルのコマンドが実行され、メニューが閉じる
3. メニュー外をクリック / Esc → 何もせず閉じる
4. 空白部分を右クリック → フォルダの背景メニューが表示される

## 各画面 / 状態の詳細

### 項目メニュー表示中

- **表示内容**: Win32 の `TrackPopupMenuEx` によるネイティブメニュー（OS テーマ準拠）
- **操作**: クリックで実行、Esc/外側クリックでキャンセル
- **遷移先**: 実行後は一覧表示のまま（削除・作成の反映は現状手動の再移動が必要）

## エラーケース

| 条件 | メッセージ / 挙動 |
|------|------------------|
| 対象がメニュー表示前に削除された | メニューを表示しない（無反応）。次の一覧操作で既存のエラー処理に委ねる |
| `QueryContextMenu` / `InvokeCommand` が失敗 | 何もしない（クラッシュさせない）。dev-notes に既知の失敗パターンを記録 |
| 「名前の変更」を選択 | シェル既定の動作に委ねる（アプリ内インライン編集は未対応。動かない場合は将来の自前実装対象） |
| エラーダイアログ（削除確認等） | シェルが自前で出すダイアログをそのまま表示（アプリは関与しない） |

## 未対応ケース

- 複数選択項目に対する一括メニュー
- 実行後の一覧の自動再読込（ファイルシステム監視は将来対応）
- ツリーノード・タブ・ペイン上の右クリックメニュー
- 自前メニュー項目（「新しいタブで開く」等）の追加 — 将来この機能の拡張として実装
