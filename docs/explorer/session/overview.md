# session 概要

## 目的・背景

作業状態（ペイン構成・タブ・各タブのパス）をアプリ終了後も維持し、開き直したときに前回の続きから使えるようにする。
ユーザ要求「閉じて開きなおしても前の状態が保存されている」を担う。

## スコープ

### 作るもの

- 状態（AppState）の JSON ファイルへの保存（終了時＋変更時の自動保存）
- 起動時の読み込み・検証・復元
- 保存データが無い/壊れている/パスが消えている場合のフォールバック

### 作らないもの

- 複数セッション（名前付きワークスペース）の管理
- ウィンドウ位置・サイズ以外の細かい UI 状態（ソート順・選択行など）の復元（将来対応）

## 制約

- 保存先は `%LOCALAPPDATA%\ExtendExprorer\session.json`
- 保存・読込は `ISessionService` のみが行う（ViewModel はサービス経由）

## 完了条件

- タブ・分割・パスを変更してアプリを終了 → 再起動で同じ画面構成が再現される
- session.json を削除/破損させても既定状態（1 ペイン 1 タブ、ホーム）で正常起動する

## 画面イメージ

（固有の画面なし。復元時に存在しないパスがあった場合のみ通知バーを表示）

```
+--------------------------------------------------+
| ⚠ 一部のフォルダが見つからなかったためホームで開きました [×] |
+--------------------------------------------------+
```
