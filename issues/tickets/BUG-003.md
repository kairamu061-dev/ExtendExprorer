# BUG-003: 一覧の背景（空白）右クリックメニューに「貼り付け」が出ない

- **State**: Fixed
- **Reported**: 2026-07-13
- **Area**: context-menu（`Services/ShellContextMenuService` の背景メニュー構築）

## Summary

一覧の空白部分の右クリックメニュー（フォルダ背景メニュー）に「貼り付け」項目が表示されない。
クリップボードにファイル（CF_HDROP）がある状態でも欠落するため、コピーはできても貼り付けができない。

## Reproduction

1. 任意のファイルを右クリック →「コピー」（アイテム側メニューには切り取り/コピーは存在）
2. 一覧の空白部分を右クリック
3. 背景メニューに「新規作成」「プロパティ」等はあるが「貼り付け」がない（クリップボード状態に関わらず）

## Expected

エクスプローラーと同様に、クリップボードにファイルがあるとき背景メニューから「貼り付け」でカレントフォルダへコピー/移動できる。

## Cause

エクスプローラーの背景メニューの「貼り付け」「ショートカットの貼り付け」「最新の情報に更新」等は、
`IShellFolder::CreateViewObject(IID_IContextMenu)` が返すフォルダ背景メニューではなく、
**DefView（シェルのフォルダビュー実装）がメニューにマージしている項目**。
本アプリは DefView をホストしていないため、CreateViewObject 経由のメニューには最初から含まれていない
（実装漏れではなくシェルの構造によるもの。新規作成・プロパティはフォルダ側 verb なので出る）。

## Fix

（対応方針）DefView のホストはコストが高すぎるため、「貼り付け」を自前項目として背景メニューへ追加する:

1. `QueryContextMenu` 後、シェル ID 範囲（〜0x7FFF）の外の ID で「貼り付け(&P)」を先頭に `InsertMenuW`
   （クリップボードに CF_HDROP がなければ無効化表示）
2. 選択されたら `OleGetClipboard` のデータオブジェクトを、フォルダの
   `CreateViewObject(IID_IDropTarget)` へ `DragEnter`→`Drop` で渡す（シェル標準の「ドロップ＝貼り付け」手法。
   コピー/移動の別はクリップボードの `Preferred DropEffect` を尊重し、無ければコピー）
3. 実際のファイルコピー・確認ダイアログ・名前衝突処理はすべてシェル側が行う
