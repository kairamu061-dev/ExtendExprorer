# session 仕様

## 機能一覧

| # | 機能名 | 説明 |
|---|--------|------|
| 1 | 自動保存 | 状態変更（タブ・ペイン・パス・比率・アクティブ変更）から 1 秒デバウンスで保存。終了時にも保存 |
| 2 | 起動時復元 | session.json を読み込み、検証のうえ AppState を復元。ウィンドウサイズ・位置も復元 |
| 3 | パス検証 | 復元時に各タブのパスの存在を確認。存在しないタブはホームパスに差し替え、通知を表示 |
| 4 | フォールバック | ファイル無し・JSON 破損・スキーマ不一致時は既定状態で起動 |

## 画面・操作フロー

1. 起動 → session.json 読込 → 検証 OK → 復元して表示（存在しないパスがあれば通知バー）
2. 起動 → 読込失敗（無し/破損）→ 既定状態（1 ペイン 1 タブ、ホーム）で表示
3. 利用中の状態変更 → デバウンス保存（ユーザ操作なし）
4. 終了 → 最終状態を同期保存してからウィンドウを閉じる

## 各画面 / 状態の詳細

### 通知バー

- **表示内容**: 「一部のフォルダが見つからなかったためホームで開きました」
- **操作**: 「×」で閉じる。10 秒で自動で消える
- **遷移先**: なし

## エラーケース

| 条件 | メッセージ / 挙動 |
|------|------------------|
| session.json が存在しない | 既定状態で起動（通知なし） |
| JSON パース失敗・スキーマ不一致 | 既定状態で起動。壊れたファイルは `session.json.bak` に退避 |
| タブのパスが存在しない | 該当タブをホームに差し替え、通知バー表示 |
| 保存時の書込失敗 | 次回変更時に再試行。アプリは継続 |

## 未対応ケース

- タブごとの履歴（戻る/進む）の復元（初期版はパスのみ復元、履歴はリセット）
- ソート順・選択行・スクロール位置の復元
