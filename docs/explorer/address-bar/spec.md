# address-bar 仕様

## 機能一覧

| # | 機能名 | 説明 |
|---|--------|------|
| 1 | パンくず表示 | アクティブタブのパスをフォルダ単位のセグメント（`[C:] › [Users] › …`）で表示。タブ切替・移動で自動更新 |
| 2 | セグメント移動 | パンくずの各フォルダ名クリックでそのフォルダへ移動（祖先は常に有効なので即移動、履歴に追加） |
| 3 | 編集モード切替 | パンくずの余白（フォルダ名以外）クリックでフルパス全選択の編集モードへ |
| 4 | パス入力移動 | 編集モードで Enter → 存在確認のうえアクティブタブを移動（ファイルパスは親フォルダへ）。履歴に追加 |
| 5 | 編集キャンセル | Esc または Enter 以外でのフォーカス喪失で編集を破棄し、パンくず表示に戻す |
| 6 | エラー表示 | 存在しないパスなら「パスが見つかりません」を 3 秒表示し、編集を継続（移動しない） |

## 画面・操作フロー

1. パンくずのフォルダ名クリック → そのフォルダへ移動（パンくず更新）
2. パンくずの余白クリック → 全選択状態で編集モードに入る
3. パスを入力して Enter → `IFileSystemService.ResolveNavigationTargetAsync` で検証 → 成功ならタブ移動、失敗ならエラー表示（編集継続）
4. Esc / Enter 以外でフォーカス喪失 → 編集破棄、パンくず表示に戻る

## 各画面 / 状態の詳細

### 表示状態

- **表示内容**: アクティブタブのフルパス（Windows 形式 `C:\...`）
- **操作**: クリックで編集状態へ
- **遷移先**: 編集状態

### 編集状態

- **表示内容**: 入力中の文字列（初期値は現在パス、全選択）
- **操作**: Enter（確定）、Esc / blur（キャンセル）
- **遷移先**: 表示状態

## エラーケース

| 条件 | メッセージ / 挙動 |
|------|------------------|
| 存在しないパス | 「パスが見つかりません」を表示、移動しない |
| ファイルのパスを入力 | 親フォルダに移動し、該当ファイル行を選択（初期版は単に親フォルダへ移動でも可） |
| アクセス不能なパス | 移動は行い、一覧側のエラー表示に委ねる |

## 未対応ケース

- 環境変数（%USERPROFILE% 等）の展開
- UNC パス（\\server\share）の特別扱い（fs がエラーを返せばエラーケース扱い）
