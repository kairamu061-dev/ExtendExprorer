# shell-icons 概要

## 目的・背景

ファイル一覧のアイコンを固定グリフ（フォルダ/ドキュメントの 2 種）から、
Windows シェルが管理する**実際の関連付けアイコン**（エクスプローラーと同じ見た目）に置き換える。
ユーザ要望（2026-07-13）「ファイルやフォルダのアイコンも Windows 標準に合わせてほしい」を担う。
project_overview の UI 方針「エクスプローラーに似せる」の一部。

## スコープ

### 作るもの

- ファイル一覧（FileListView）の行アイコンをシェルアイコン（16px 相当）で表示
- 拡張子単位のキャッシュ（同じ種類のファイルはアイコンを 1 回だけ取得）
- .exe / .ico / .lnk はファイル個別のアイコン（パス単位でキャッシュ）
- 取得完了までは従来のグリフを表示し、取得後に差し替え（一覧表示をブロックしない）

### 作らないもの

- folder-tree のノードアイコンのシェルアイコン化（将来対応。現状のホーム/ドライブ/フォルダグリフを維持）
- オーバーレイアイコン（ショートカット矢印・共有マーク等）
- サムネイル表示（画像のプレビュー等）
- 特殊フォルダの個別アイコン（Documents 等。フォルダは汎用アイコンで統一）

## 制約

- AOT 互換必須: アイコン取得は `LibraryImport`（SHGetFileInfoW / GDI）のみで実装
- アイコン取得・ピクセル変換は UI スレッドで行わない（Task.Run）。UI スレッドでは WriteableBitmap 生成のみ
- キャッシュはプロセス生存中のみ（永続化しない）。メモリ増を +数 MB 以内に抑える

## 完了条件

- .txt / .pdf / .zip / .exe / フォルダ等がエクスプローラーと同じアイコンで表示される
- 大量ファイルのフォルダでもスクロールが引っかからない
- アイコン取得失敗時は従来グリフのまま（クラッシュ・空白にしない）

## 画面イメージ

```
+--------------------------------------+
| 名前            更新日時    種類  サイズ |
| [📁] Documents  2026/07/01 フォルダ —  |
| [📄] readme.txt 2026/07/02 TXT   1 KB |  ← 拡張子の関連付けアイコン
| [⚙] tool.exe    2026/07/03 EXE  2 MB |  ← exe 個別のアイコン
+--------------------------------------+
```
