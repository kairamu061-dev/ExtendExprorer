## E2Eテスト（手動）

| ID | テスト名 | 前提条件 | 手順 | 期待結果 | ステータス |
|----|---------|---------|------|---------|-----------|
| E-01 | 起動時ホーム表示 | Windows 11 で exe 起動 | 起動する | ユーザプロファイル直下の一覧が表示され、ツールバーにパスが出る | [x] |
| E-02 | フォルダ移動 | E-01 | 任意のフォルダ行をダブルクリック | そのフォルダの内容に切り替わり、パス表示が更新される | [x] |
| E-03 | 上へ | E-02 | 「上へ」ボタンを押す | 親フォルダに戻る。ドライブルートでは「上へ」が無効化される | [x] |
| E-04 | 戻る / 進む | E-02 | 「戻る」→「進む」を押す | 履歴どおりに移動し、端ではボタンが無効化される | [x] |
| E-05 | ソート | E-01 | 「名前」ヘッダを 2 回クリック、次に「サイズ」をクリック | 昇順→降順に切替わる。列を変えると昇順から。常にフォルダが先頭グループ | [x] |
| E-06 | アクセス拒否 | E-01 | `C:\System Volume Information` 等へ移動を試みる | 一覧領域に「アクセスが拒否されました」が表示され、アプリは落ちず「戻る」で復帰できる | [x] |
| E-07 | 表示書式 | E-01 | 一覧の各列を目視確認 | 日時 `YYYY/MM/DD HH:mm`、フォルダのサイズは「—」、ファイルは KB/MB/GB 表記、種類は拡張子大文字（なしは「ファイル」） | [x] |
| E-08 | ソートのリセット | E-05 | サイズ降順にした状態で別フォルダへ移動する | 移動先では名前昇順（既定）に戻っている | [x] |
| E-09 | 隠し/システムの薄色表示 | E-01 | ホームや `C:\` で `NTUSER.DAT` / `$Recycle.Bin` / `pagefile.sys` 等を目視 | 隠し・システム属性の行だけ薄色（減光）で表示される | [x] |
| E-10 | UIA 行名 | E-01 | UIA インスペクタ等で一覧行の Name を確認 | 行の Name がファイル名になっている（クラス名ではない） | [x] |

E-01〜E-07 は 2026-07-05 の動作確認（run 28728613563, commit `2da7ec4`）、E-08〜E-10 は同日の再確認（run 28734759376, commit `5077431`）で合格。
