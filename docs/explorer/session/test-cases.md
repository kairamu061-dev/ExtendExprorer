## E2Eテスト（手動）

| ID | テスト名 | 前提条件 | 手順 | 期待結果 | ステータス |
|----|---------|---------|------|---------|-----------|
| E-01 | 基本復元 | 複数タブ・分割・各パスを設定 | アプリを終了→再起動 | 同じペイン構成・タブ・パスが再現される | [ ] |
| E-02 | ウィンドウ位置/サイズ | ウィンドウを移動・リサイズ | 終了→再起動 | 同じ位置・サイズで開く | [ ] |
| E-03 | アクティブ状態 | 特定ペイン/タブをアクティブに | 終了→再起動 | 同じペイン・タブがアクティブ | [ ] |
| E-04 | 分割比率 | スプリッターで比率を変更 | 終了→再起動 | 同じ比率で復元される | [ ] |
| E-05 | 不存在パス | 保存後にそのフォルダを削除/リネーム | 再起動 | 該当タブがホームに差し替わり、通知バーが出る（10 秒で消える） | [ ] |
| E-06 | ファイル無し | session.json を削除 | 再起動 | 既定状態（1 ペイン 1 タブ・ホーム）で通知なく起動 | [ ] |
| E-07 | 破損 JSON | session.json を壊す | 再起動 | 既定状態で起動し、壊れたファイルが session.json.bak に退避される | [ ] |
| E-08 | 自動保存 | 操作後に強制終了（保存待たず） | 再起動 | 直近（デバウンス済み）の状態が概ね復元される | [ ] |
| E-09 | AOT 安定性 | 正式(AOT)ビルド | E-01〜E-08 | クラッシュ・WER なし（AOT で JSON ソース生成が動作） | [ ] |

## ステータス凡例

- `[ ]` 未実施
- `[~]` 実施中
- `[x]` 合格
- `[!]` 不合格
