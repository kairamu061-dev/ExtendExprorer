# BUG-001: Native AOT ビルドが起動時にクラッシュする

- **State**: In Progress
- **Reported**: 2026-07-05
- **Area**: ビルド構成（src/ExtendExprorer/ExtendExprorer.csproj / .github/workflows/build.yml の aot フレーバー）

## Summary

CI の Native AOT 実験ビルド（`ExtendExprorer-win-x64-aot-experiment`）がウィンドウ表示前にクラッシュし、一度も起動できない。通常ビルドは正常。

## Reproduction

1. run `28734759376`（commit `5077431`）の AOT 成果物をダウンロード・展開
2. `ExtendExprorer.exe` を起動
3. ウィンドウが表示されずプロセス終了（再現率 2/2）
   - 終了コード `0xC000027B`（stowed exception）
   - 障害モジュール `Microsoft.UI.Xaml.dll` 3.1.6.0、WER P7=`80004002`（E_NOINTERFACE）

## Expected

通常ビルドと同様に起動し、file-list の全機能が動作する。

## Cause

（推定・検証セッションの所見に基づく）Windows App SDK 1.6 系（Microsoft.UI.Xaml 3.1.6.0）の CsWinRT AOT 用インターロップ生成が不十分で、XAML 初期化中のジェネリック WinRT インターフェース問い合わせが E_NOINTERFACE で失敗する。コード側は `{x:Bind}` のみ・ViewModel は partial で問題見当たらず、フレームワーク側の AOT 対応成熟度が主因と判断。

## Fix

- Microsoft.WindowsAppSDK を 1.6.* → 1.7.*（AOT 対応が成熟した系列）へ更新
- `CsWinRTAotOptimizerEnabled` を明示的に有効化し、`CsWinRTAotWarningLevel=2` で AOT 非互換をビルド警告として可視化

- **Fix commit**: `b596ecf`

## Notes

- ビルド成功は CI で確認済みでも、起動可否はユーザ環境での再確認が必要（本チケットは検証合格まで Closed にしない）
- 再検証手順は tmp/確認/ の依頼ファイルを参照（起動→E-01〜E-07 相当のスモーク→メモリ計測）

### 追加対応（フィードバック反映）: CommunityToolkit.Mvvm の AOT 非互換を解消

フィードバックの容疑リストのうち「CommunityToolkit.Mvvm の AOT 対応」に着手。

- **内容**: フィールドベースの `[ObservableProperty]`（WinRT/AOT では **MVVMTK0045** 非互換）を全 ViewModel で手書きプロパティ（`SetProperty` 呼び出し）へ置換
  - 対象: `TabViewModel`（Path/ErrorMessage/IsLoading/SortColumn/SortAscending）, `PaneViewModel`（ActiveTab）, `MainViewModel`（Layout/ActivePane）, `SplitNodeViewModel`（Ratio）
  - `partial void OnPathChanged` は `Path` セッター内の `OnPropertyChanged(nameof(Title))` に統合
- **csproj**: `AllowUnsafeBlocks=true` を追加（CsWinRT1030: ジェネリック WinRT インターフェースのマーシャリング生成に必要）
- **未検証**: ローカルは Linux/WSL のため AOT publish 不可。次回 CI の AOT 成果物でユーザ環境の再検証が必要。IL2026/IL3050 警告の精査は継続
- State は **In Progress** のまま（再検証合格まで Fixed にしない）
