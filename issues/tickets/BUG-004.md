# BUG-004: ダブルクリックで既定アプリがあるのに「開く方法」チューザーが出る

- **State**: Open
- **Reported**: 2026-07-19
- **Area**: file-list（`FileListView.OnItemDoubleTapped` → `ShellExecuteW`）

## Summary

既定アプリが正しく関連付けられている拡張子（.txt = サクラエディタ）をダブルクリックしても直接開かず、
「アプリを選択して .txt ファイルを開く」チューザーが表示される（ダイアログ内には既定アプリが表示されている）。

## Reproduction

1. 既定アプリが設定済みの .txt をダブルクリック
2. 既定アプリで直接開かれず、「開く方法」チューザーが表示される（再現率 100%）
3. 参考: 関連付けのない拡張子（E-21）ではチューザーが出るのが正しい挙動で、こちらは合格

## Expected

エクスプローラーのダブルクリックと同様、既定アプリで直接開く。

## Cause

`ShellExecuteW(hwnd, null, path, null, dir, SW_SHOWNORMAL)` の呼び出しが実質 openas 相当の挙動になっている
（E-21 も同じ経路でチューザーが出ることから、verb 解決がファイルの既定 verb に到達していない）。
一方、同じファイルをシェルコンテキストメニューの「開く」（IContextMenu 経由）で開くと正しく既定アプリが起動する
（context-menu E-02 合格）ため、パス文字列ベースの ShellExecuteW ではなく PIDL ベースの起動なら正しく解決される。

## Fix

`ShellExecuteExW`＋`SEE_MASK_INVOKEIDLIST`（PIDL 指定・verb 省略=既定 verb）へ切替。
エクスプローラーのダブルクリックと同じ経路（アイテムの既定コンテキストメニュー verb の実行）になり、
動作実績のある context-menu E-02 と同じ解決パスを通る。
