# file-list 仕様

## 機能一覧

| # | 機能名 | 説明 |
|---|--------|------|
| 1 | 一覧表示 | 指定パスの内容を名前・更新日時・種類・サイズで表示。フォルダ先頭、次にファイル。隠し・システム属性の項目も常に表示するが、行を薄色（減光）で表示して区別する |
| 2 | ソート | 列ヘッダクリックで昇順/降順を切替（既定: 名前昇順。フォルダ先頭は維持）。ソート状態はフォルダ単位で、別フォルダへ移動すると既定（名前昇順）にリセットされる |
| 3 | フォルダ移動 | フォルダ行のダブルクリックでそのフォルダへ移動 |
| 3b | ファイルを開く | ファイル行のダブルクリックで拡張子の既定アプリで開く（シェルの関連付けに従う。2026-07-13 ユーザ要望） |
| 4 | 履歴移動 | 「戻る」「進む」でタブの履歴を移動、「上へ」で親フォルダへ |
| 5 | エラー表示 | 列挙失敗時、一覧領域にエラーメッセージを表示 |
| 6 | 複数選択 | Shift（範囲）/ Ctrl（個別追加）で複数選択（ListView Extended。2026-07-19 ユーザ要望）。右クリックメニュー・コピー・D&D は選択全項目が対象 |
| 7 | リネーム | 選択済み項目（単一選択）をもう一度クリックし、ダブルクリック間隔を空けるとインライン編集開始（エクスプローラーと同じ操作。2026-07-19 ユーザ要望）。Enter=確定 / Esc=取消 / フォーカス喪失=確定。実リネームはシェル(IFileOperation)で、確定後に一覧を再読込（ソート維持） |

## 画面・操作フロー

1. タブのパスが変わる（初期表示・移動・履歴移動）→ `IFileSystemService.ListAsync` で列挙し一覧を再描画
2. フォルダ行をダブルクリック → タブの履歴に追加してパス変更 → 1 に戻る
   （ファイル行のダブルクリックは既定アプリで開く。関連付けがない場合はシェルの既定動作＝「このファイルを開く方法」ダイアログ）
3. 「上へ」→ 親パスへ移動（ルートでは無効化）
4. 「戻る」「進む」→ `historyIndex` を移動して再描画（端では無効化）

## 各画面 / 状態の詳細

### 一覧テーブル

- **表示内容**: 種類は拡張子の大文字表記（フォルダは「フォルダ」、拡張子なしは「ファイル」）。サイズはフォルダは「—」、ファイルは KB/MB/GB 表記。更新日時は `YYYY/MM/DD HH:mm`。隠し・システム属性の行は不透明度を下げて薄色表示
- **操作**: 行クリックで選択（ハイライトのみ）、ヘッダクリックでソート
- **遷移先**: フォルダのダブルクリックで同一タブ内パス変更

### エラー状態

- **表示内容**: 一覧領域中央に「アクセスが拒否されました」「パスが見つかりません」等
- **操作**: 「上へ」「戻る」は引き続き使用可能
- **遷移先**: 移動操作で正常状態に復帰

### リネームの補足

- ファイルは拡張子を除いた部分を選択した状態で編集開始（フォルダは全選択）
- 空文字・変更なしは取消扱い。不正な文字・同名衝突のダイアログはシェルに委ねる
- ダブルクリック（開く）とはダブルクリック間隔（OS 設定 `GetDoubleClickTime`）で区別する

## エラーケース

| 条件 | メッセージ / 挙動 |
|------|------------------|
| UnauthorizedAccessException | 「アクセスが拒否されました」 |
| DirectoryNotFoundException | 「パスが見つかりません」 |
| その他の IO 例外 | 「読み込みに失敗しました: {メッセージ}」 |

## 未対応ケース

- 隠しファイルの表示/非表示切替（初期版は常に表示＋薄色。切替は将来対応）
- 大量ファイル（1 万件超）の仮想スクロール（性能問題が出たら対応）
