# context-menu 概要

## 目的・背景

ファイル一覧の項目を右クリックしたとき、Windows 標準（シェル）のコンテキストメニューを表示する。
ユーザ要望（2026-07-12）「右クリックメニューがない。Windows のコンテキストメニューでお願いします」を担う。
自前メニューではなくシェルのメニューを出すことで、開く／コピー／削除／プロパティ／送る／
インストール済みアプリの拡張項目まで、エクスプローラーと同じ操作性をそのまま提供する。

## スコープ

### 作るもの

- 一覧項目の右クリックで、その項目に対する **Windows シェルのコンテキストメニュー**を表示・実行
- 一覧の空白部分の右クリックで、表示中フォルダの**背景メニュー**（新規作成・貼り付け・プロパティ等）を表示・実行
- 「送る」「プログラムから開く」などの動的サブメニューへの対応（IContextMenu2/3 のメッセージ転送）

### 作らないもの

- 自前のカスタムメニュー項目の追加（将来対応）
- 複数選択への対応（一覧が単一選択のため。tabs/file-list 側で複数選択を導入したら拡張）
- ツリー（folder-tree）ノード上での右クリックメニュー

## 制約

- **Native AOT 互換必須**: COM 相互運用はランタイムの組み込みマーシャリング（AOT 非対応）を使わず、
  .NET 8 の**ソース生成 COM**（`[GeneratedComInterface]` / `ComWrappers`）と P/Invoke（`LibraryImport`）のみで実装する
- メニュー表示・実行は UI（STA）スレッド上で行う
- 追加の NuGet 依存を増やさない（CsWin32 等は使わず必要な宣言のみ手書き）

## 完了条件

- 項目右クリックでエクスプローラー同等のメニューが出て、「開く」「コピー」「削除」「プロパティ」が動作する
- 「送る」サブメニューが正しく展開・実行できる
- 空白右クリックで背景メニューが出て「新規作成」「貼り付け」が動作する
- AOT ビルドで警告ゼロ・実機でクラッシュしない

## 画面イメージ

```
+--------------------------------------+
| ファイル一覧                          |
|   document.txt   ◀ 右クリック         |
|     +----------------------------+   |
|     | 開く                        |   |
|     | プログラムから開く        ▶ |   |
|     | 送る                      ▶ |   |
|     | コピー / 切り取り / 削除     |   |
|     | プロパティ                  |   |
|     +----------------------------+   |
+--------------------------------------+
  （内容は OS・インストール済みアプリに依存）
```
